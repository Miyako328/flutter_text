# 月下远征放置玩法设计草案

## 1. 设计目标

本系统最终目标是承载《月下，她与她的长夜》的大世界放置探索玩法，但第一阶段只开放一个区域与一个探索主题。

当前阶段不做肉鸽事件、不做战斗、不做复杂角色编队，先验证最小闭环：

```text
大世界地图 -> 进入区域 -> 选择探索路线 -> 等待放置完成 -> 领取收集物 -> 局外养成 -> 解锁下一关
```

第一阶段开放内容：

```text
区域：南境裂谷与暮色镇
第一关：暮色镇周边探索
目标：收集玛克莱线索，逐步解锁“玛克莱遗址深入”
```

设计原则：

- 使用大世界架构，但只填充一个区域，避免未来推倒重来。
- 玩法先纯放置，先跑通时间推进、奖励、收集、养成。
- 阿特拉斯不在第一关直接出现，而是通过线索收集逐渐逼近。
- 所有数值可调，第一版先短时长测试，后续再拉长。

## 2. 当前区域定位

### 2.1 区域

```text
区域名：南境裂谷与暮色镇
主题：苏醒、暮色镇协会、血雾、玛克莱旧印、阿特拉斯残响
开放状态：第一阶段开放
```

区域功能：

- 作为大世界第一块可玩区域。
- 承载暮色镇、诺希尔森林外缘、玛克莱遗址相关探索。
- 通过多轮放置收集线索，解锁后续关卡。

### 2.2 当前关卡

```text
关卡名：暮色镇周边探索
关卡类型：放置收集
直接目标：收集关键线索
长期目标：解锁“玛克莱遗址深入”
```

当前关卡不包含 Boss。

后续关卡规划：

```text
第二关：玛克莱遗址深入
目标：进入遗址，确认王城残壁、壁画、深蓝魔力炉等关键点。

第三关：阿特拉斯讨伐
目标：在收集足够线索与削弱条件后，压制或终结阿特拉斯残响。
```

重要规则：

- “玛克莱遗址深入”首次出现需要完成第一关的线索收集目标。
- 出现后，该关卡会在地图与区域页中常驻显示，不会因为挑战完成而消失。
- 但每次进入仍需要消耗线索门票。
- 线索不只是一次性解锁条件，也是后续反复进入 Boss/深入关卡的入场资源。

## 3. 第一关核心收集

第一关的核心进度由五类线索构成。

### 3.1 暮色镇情报

含义：

- 来自暮色镇协会、酒馆、镇民、巡夜人、失踪记录的基础情报。
- 代表玩家知道“该去哪里找”。

来源：

- 暮色镇调查。
- 协会旧记录。
- 酒馆传闻。
- 镇外失踪报告。

用途：

- 解锁更深入的探索路线。
- 提高探索基础收益。
- 作为下一关解锁条件之一。
- 后续可作为进入“玛克莱遗址深入”的门票材料之一。

建议目标：

```text
暮色镇情报 0 / 5
```

### 3.2 死徒残痕

含义：

- 低阶死徒、异常血迹、拖行痕迹、夜间袭击痕迹。
- 代表阿特拉斯残留影响仍在区域内活动。

来源：

- 森林外缘巡查。
- 镇外旧路。
- 被袭击的兽尸或商队残骸。

用途：

- 定位高危区域。
- 解锁玛克莱遗址外围探索。
- 提高后续 Boss 关前置识别度。
- 后续可作为进入“玛克莱遗址深入”的门票材料之一。

建议目标：

```text
死徒残痕 0 / 8
```

### 3.3 玛克莱旧印

含义：

- 黑红纹章、旧帝国石片、壁画碎屑、祭祀符文拓片。
- 代表玩家逐渐确认玛克莱遗址和旧帝国历史落点。

来源：

- 旧路遗迹搜寻。
- 遗址外围残片。
- 地下石道或旧祭坛残砖。

用途：

- 提升玛克莱档案等级。
- 解锁遗址深入。
- 解锁后续壁画与鲜血日冕相关文本。
- 后续可作为进入“玛克莱遗址深入”的核心门票材料。

建议目标：

```text
玛克莱旧印 0 / 4
```

### 3.4 血雾样本

含义：

- 夜间血雾、腐败血液凝块、异常魔力雾、死徒消散残留。
- 代表玩家开始接触阿特拉斯灵魂魔法与血雾影响。

来源：

- 森林外缘巡查。
- 旧路遗迹搜寻。
- 高等级血契掌控提高获取效率。

用途：

- 后续用于削弱 Boss 关污染或异常压力。
- 升级血契掌控。
- 作为下一关解锁条件之一。
- 后续可作为进入“玛克莱遗址深入”的门票材料之一。

建议目标：

```text
血雾样本 0 / 6
```

### 3.5 阿特拉斯残响

含义：

- 最稀有线索。
- 代表玩家真正捕捉到阿特拉斯的灵魂残留或方向。

来源：

- 旧路遗迹搜寻低概率获得。
- 玛姬残响等级提高概率。
- 后续可由特殊探索路线稳定产出。

表现：

- 血雾里的王冠幻影。
- 梦中低语。
- 旧王座方向的呼唤。
- 深蓝魔力炉异常共鸣。

用途：

- 解锁阿特拉斯相关关卡。
- 削弱后续 Boss 初始强度。
- 解锁特殊剧情文本。
- 后续可作为进入“玛克莱遗址深入”或“阿特拉斯讨伐”的高级门票材料。

建议目标：

```text
阿特拉斯残响 0 / 3
```

## 4. 探索路线

第一关先开放三条放置路线。每条路线有不同耗时、产出与定位。

### 4.1 暮色镇调查

定位：

- 安全、短耗时、基础线索。

基础耗时：

```text
3 分钟
```

基础产出：

```text
金币 +10
暮色镇情报 +1 到 +2
暮色镇声望 +1
```

可能日志：

```text
你在冒险者协会旧柜台里找到了几份失踪记录。记录上的路线都指向镇外同一片森林。
```

适合阶段：

- 前期快速建立情报基础。
- 升级暮色镇协会。

### 4.2 森林外缘巡查

定位：

- 中耗时、普通风险、血迹与死徒线索。

基础耗时：

```text
5 分钟
```

基础产出：

```text
金币 +8
死徒残痕 +1 到 +3
血雾样本 +0 到 +1
血痕残片 +0 到 +1
```

可能日志：

```text
队伍在腐叶下发现了拖行痕迹。那不是普通野兽留下的痕迹，血味太冷，也太旧。
```

适合阶段：

- 中期收集死徒残痕和血雾样本。
- 升级希娅战斗熟练度与血契掌控。

### 4.3 旧路遗迹搜寻

定位：

- 长耗时、高价值、旧帝国线索与稀有残响。

基础耗时：

```text
8 分钟
```

基础产出：

```text
金币 +6
玛克莱旧印 +1 到 +2
旧帝国碎片 +0 到 +1
阿特拉斯残响 小概率 +1
```

可能日志：

```text
一块黑红色石片从泥土里露出来，边缘仍残留着玛克莱旧纹章的刻痕。
```

稀有日志：

```text
夜色最深时，队伍听见了一个像从王座下传来的声音。它没有说话，只是在血雾里短暂地笑了一声。
```

适合阶段：

- 后期推进玛克莱旧印与阿特拉斯残响。
- 升级玛克莱档案与玛姬残响。

## 5. 奖励体系

奖励分为四层：基础资源、收集资源、养成资源、解锁资源。

### 5.1 基础资源

```text
金币
暮色镇声望
小队经验
```

第一版可以先不做小队经验，只保留金币与暮色镇声望。

用途：

- 金币：通用升级资源。
- 暮色镇声望：升级暮色镇协会，后续解锁更高等级委托。

### 5.2 收集资源

```text
暮色镇情报
死徒残痕
玛克莱旧印
血雾样本
阿特拉斯残响
```

用途：

- 推进第一关主目标。
- 解锁第二关。
- 参与部分局外养成。

### 5.3 养成资源

```text
血痕残片
旧帝国碎片
日冕微光
深蓝炉灰
```

第一版建议只做：

```text
血痕残片
旧帝国碎片
```

后续资源用途：

- 血痕残片：升级血契掌控、希娅战斗熟练度。
- 旧帝国碎片：升级玛克莱档案、玛姬残响。
- 日冕微光：后续特殊养成资源。
- 深蓝炉灰：后续 Boss 或遗址深入资源。

### 5.4 解锁资源

解锁资源更像永久发现，不一定作为可消耗货币。

```text
遗址坐标
残破壁画拓片
阿特拉斯灵魂刻印
深蓝魔力炉记录
```

第一版可以先不单独建成道具，而是由收集进度自动触发。

示例：

```text
暮色镇情报、死徒残痕、玛克莱旧印、血雾样本、阿特拉斯残响全部达到目标后：
解锁“玛克莱遗址深入”。
```

## 6. 局外养成

第一版保留五个养成项。

### 6.1 暮色镇协会等级

定位：

- 区域基础设施。

升级消耗：

```text
金币
暮色镇声望
```

效果：

```text
每级金币收益 +10%
每级暮色镇调查耗时 -3%
每 2 级暮色镇声望收益 +1
```

第一版简化公式：

```text
暮色镇调查金币 = 10 + 暮色镇协会等级 * 2
```

### 6.2 玛克莱档案等级

定位：

- 区域档案树。

升级消耗：

```text
玛克莱旧印
旧帝国碎片
```

效果：

```text
每级玛克莱旧印收益提高
解锁玛克莱相关日志文本
提高旧路遗迹搜寻额外发现概率
```

第一版简化公式：

```text
旧路遗迹搜寻有概率额外获得 玛克莱旧印 +1
概率 = 玛克莱档案等级 * 5%
```

### 6.3 希娅战斗熟练度

定位：

- 角色/队伍战斗向成长。

升级消耗：

```text
金币
血痕残片
```

效果：

```text
森林外缘巡查耗时减少
死徒残痕收益提高
后续战斗关降低损耗
```

第一版简化公式：

```text
所有探索耗时 = 基础耗时 * (1 - 希娅战斗熟练度 * 0.03)
最低不低于基础耗时的 70%
```

### 6.4 血契掌控

定位：

- 风险控制与血雾解析。

升级消耗：

```text
血痕残片
血雾样本
```

效果：

```text
血雾样本收益提高
降低后续污染或暴走类代价
小幅提高阿特拉斯残响发现率
```

第一版简化公式：

```text
森林外缘巡查获得血雾样本概率 + 血契掌控等级 * 5%
```

### 6.5 玛姬残响

定位：

- 特殊同行者/旧日记忆树。

升级消耗：

```text
旧帝国碎片
阿特拉斯残响
```

效果：

```text
提高阿特拉斯残响发现率
解锁特殊日志文本
后续 Boss 关削弱阿特拉斯初始强度
```

第一版简化公式：

```text
旧路遗迹搜寻获得阿特拉斯残响概率 = 5% + 玛姬残响等级 * 3%
```

## 7. 关卡解锁与门票规则

当以下条件全部满足时，解锁第二关“玛克莱遗址深入”：

```text
暮色镇情报 >= 5
死徒残痕 >= 8
玛克莱旧印 >= 4
血雾样本 >= 6
阿特拉斯残响 >= 3
```

解锁提示：

```text
玛克莱遗址坐标已确认。
阿特拉斯残响已被捕捉。
新关卡解锁：玛克莱遗址深入。
```

### 7.1 常驻显示

“玛克莱遗址深入”解锁后会常驻显示在区域页与地图入口中。

关卡状态建议分为：

```text
locked      未解锁
available   已解锁，可进入
running     进行中
finished    可领取
```

解锁后，即使完成过该关卡，也不会回到 locked。

### 7.2 门票消耗

每次进入“玛克莱遗址深入”都需要消耗一组线索门票。

第一版建议门票：

```text
暮色镇情报 -1
死徒残痕 -2
玛克莱旧印 -1
血雾样本 -1
```

如果后续将“阿特拉斯残响”定位为更稀有资源，可以不在第二关消耗它，而是留给第三关“阿特拉斯讨伐”。

第三关建议门票：

```text
阿特拉斯残响 -1
玛克莱旧印 -2
血雾样本 -2
```

### 7.3 解锁进度与门票库存分离

为了避免玩家觉得“升级或进关消耗线索后，解锁进度倒退”，建议区分两个概念：

```text
历史发现量：用于判断关卡是否解锁，只增不减。
当前持有量：作为门票和养成消耗，会增减。
```

例如：

```text
玛克莱旧印
- 历史发现：4 / 4
- 当前持有：2
```

这样“玛克莱遗址深入”一旦解锁，就不会因为消耗门票而重新锁上。

## 8. 放置推进规则

第一版不要持续写数据库。

后端只存：

```text
探索路线
开始时间
预计完成时间
状态
```

打开 App 时动态计算：

```text
剩余时间 = 预计完成时间 - 当前时间
进度 = 已经过时间 / 总耗时
```

状态：

```text
idle      空闲
running   探索中
finished  可领取
claimed   已领取
```

同一时间建议只允许一个探索进行，降低第一版复杂度。

## 9. Flame 联动方式

第一阶段 Flame 只做表现层，不做奖励逻辑。

### 9.1 Flutter 负责

```text
选择探索路线
展示剩余时间
展示奖励
升级局外养成
调用后端接口
```

### 9.2 后端负责

```text
探索开始
探索完成判断
奖励结算
资源写入
养成升级
关卡解锁
```

### 9.3 Flame 负责

```text
区域路线图表现
小队图标移动
节点高亮
探索完成动画
后续 Boss 关演出
```

第一版 Flame 路线：

```text
暮色镇 -> 森林外缘 -> 旧路遗迹 -> 玛克莱遗址方向
```

纯放置阶段可以先用 Flutter 静态路线图代替 Flame。等放置闭环通了，再替换成 Flame 动画。

## 10. 数据库草案

### 10.1 区域表

```sql
CREATE TABLE idle_regions (
  id INT PRIMARY KEY AUTO_INCREMENT,
  region_key VARCHAR(100) NOT NULL UNIQUE,
  name VARCHAR(100) NOT NULL,
  description TEXT,
  unlocked TINYINT NOT NULL DEFAULT 1,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 10.2 关卡表

```sql
CREATE TABLE idle_stages (
  id INT PRIMARY KEY AUTO_INCREMENT,
  region_id INT NOT NULL,
  stage_key VARCHAR(100) NOT NULL UNIQUE,
  name VARCHAR(100) NOT NULL,
  description TEXT,
  unlocked TINYINT NOT NULL DEFAULT 0,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (region_id) REFERENCES idle_regions(id)
);
```

### 10.3 探索路线表

```sql
CREATE TABLE idle_routes (
  id INT PRIMARY KEY AUTO_INCREMENT,
  stage_id INT NOT NULL,
  route_key VARCHAR(100) NOT NULL UNIQUE,
  name VARCHAR(100) NOT NULL,
  duration_seconds INT NOT NULL,
  description TEXT,
  unlocked TINYINT NOT NULL DEFAULT 1,
  FOREIGN KEY (stage_id) REFERENCES idle_stages(id)
);
```

### 10.4 玩家资源表

第一版单人自用，可以先不做 user_id。

```sql
CREATE TABLE idle_resources (
  resource_key VARCHAR(100) PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  amount INT NOT NULL DEFAULT 0
);
```

如果要支持“历史发现量”和“当前持有量”分离，建议改为：

```sql
CREATE TABLE idle_resources (
  resource_key VARCHAR(100) PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  amount INT NOT NULL DEFAULT 0,
  total_earned INT NOT NULL DEFAULT 0
);
```

其中：

```text
amount：当前持有量，可被门票和升级消耗。
total_earned：历史累计获得量，只增不减，用于关卡解锁判断。
```

### 10.5 局外养成表

```sql
CREATE TABLE idle_upgrades (
  upgrade_key VARCHAR(100) PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  level INT NOT NULL DEFAULT 0,
  description TEXT
);
```

### 10.6 探索记录表

```sql
CREATE TABLE idle_expeditions (
  id INT PRIMARY KEY AUTO_INCREMENT,
  route_id INT NOT NULL,
  status VARCHAR(30) NOT NULL DEFAULT 'running',
  started_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  finish_at TIMESTAMP NOT NULL,
  claimed_at TIMESTAMP NULL,
  result_title VARCHAR(100),
  result_content TEXT,
  FOREIGN KEY (route_id) REFERENCES idle_routes(id)
);
```

### 10.7 探索奖励表

```sql
CREATE TABLE idle_expedition_rewards (
  id INT PRIMARY KEY AUTO_INCREMENT,
  expedition_id INT NOT NULL,
  resource_key VARCHAR(100) NOT NULL,
  amount INT NOT NULL,
  FOREIGN KEY (expedition_id) REFERENCES idle_expeditions(id) ON DELETE CASCADE
);
```

## 11. 接口草案

第一版需要四个接口。

### 11.1 查看状态

```text
GET /moonlit/api/idle_state.php
```

返回：

### 11.2 开始探索

```text
POST /moonlit/api/start_idle.php
route_key=twilight_investigation
```

规则：

- 如果已有 running/finished 未领取探索，则不允许新开。
- 根据路线基础耗时和养成等级计算完成时间。
- 创建 idle_expeditions 记录。

### 11.3 领取奖励

```text
POST /moonlit/api/claim_idle.php
expedition_id=1
```

规则：

- 当前时间小于 finish_at 时不可领取。
- 已领取不可重复领取。
- 根据路线、养成等级、随机小浮动生成奖励。
- 写入 idle_resources。
- 写入 idle_expedition_rewards。
- 增加资源 amount 与 total_earned。
- 检查是否解锁下一关。

### 11.4 升级养成

```text
POST /moonlit/api/upgrade_idle.php
upgrade_key=maclay_archive
```

规则：

- 检查资源是否足够。
- 扣除资源。
- 提升等级。

### 11.5 进入已解锁关卡

```text
POST /moonlit/api/start_stage.php
stage_key=maclay_ruins_deep
```

规则：

- 检查关卡是否已经通过 total_earned 条件解锁。
- 检查门票资源 amount 是否足够。
- 扣除门票。
- 创建该关卡的探索记录。

```text
旧帝国碎片：0-1
阿特拉斯残响：5% 基础概率
```

### 12.2 养成上限

第一版建议每个养成先限制到 5 级。

```text
暮色镇协会 Lv0-5
玛克莱档案 Lv0-5
希娅战斗熟练度 Lv0-5
血契掌控 Lv0-5
玛姬残响 Lv0-5
```

### 12.3 解锁节奏

目标是让玩家在 1-2 天轻度使用内解锁第二关。

测试阶段可以把耗时缩短到：

```text
暮色镇调查：30 秒
森林外缘巡查：45 秒
旧路遗迹搜寻：60 秒
```

等玩法稳定后，再恢复正式耗时。

## 13. 可优化点清单

后续可以重点讨论这些问题：

- 第一关收集目标是否过多，是否需要从五类降到四类。
- 阿特拉斯残响是否应该纯随机，还是加入保底机制。
- 局外养成是否要消耗核心线索，避免玩家觉得“拿解锁进度去升级很亏”。
- 暮色镇声望是否应该独立存在，还是并入金币。
- 是否需要允许多个探索队列同时进行。
- Flame 是否第一版就接入，还是等后端闭环完成后再加。
- 是否需要将日志文本也放数据库，方便后续扩展。
- 是否要保留“污染”作为隐藏失败压力，还是第一版完全不做惩罚。

## 14. 当前建议

第一版先做最小闭环：

```text
三条探索路线
九种资源
五个养成项
一个解锁目标
四个接口
一个 Flutter 页面
```

暂不做：

```text
肉鸽事件
Boss 战
多角色编队
世界影响
多区域并行
复杂失败惩罚
```

这样既能验证“放置 + 收集 + 养成”的基础乐趣，也不会把第一版做得太重。
